# 3 — Instance import/export execution

**Status:** ✅ DONE

## What exists

- **Executor** (`executor.rb`): Orchestrates import/validate/export cycle
- **FormatAdapter** (`executor/format_adapter.rb`): Pluggable registry for format-specific I/O.
  Built-in adapters (csv, xml) are resolved lazily via constant autoload — no
  explicit `require` needed.
- **CsvAdapter** (`executor/csv_adapter.rb`): CSV import with column mapping, export with
  auto-detected headers from compiled class attributes
- **XmlAdapter** (`executor/xml_adapter.rb`): XML import/export via moxml. Delegates
  per-record (de)serialization to lutaml-model's XML mapping auto-generated by
  `ModelCompiler.apply_xml_mapping`.
- **ConditionEvaluator** (`executor/condition_evaluator.rb`): Collection validation —
  count comparisons (`count >= N`) and block predicates (`all? { |i| ... }`,
  `any? { |i| ... }`) with safe literal parsing (no `eval`).

## XML adapter design

The adapter uses the lutaml-model XML mapping generated on each compiled class
by `ModelCompiler`:

```ruby
# LML model definition
class Product {
  attribute id { type String }
  attribute name { type String }
}
```

→ ModelCompiler generates a `Serializable` subclass with `xml do |m|
m.root("Product"); m.map_element("id", to: :id); ... end`

→ XmlAdapter.import calls `target_class.from_xml(element.to_xml)` per record
→ XmlAdapter.export wraps `inst.to_xml` output under a configurable root

Import shape (from LML `import { xml "..." { map_to X; where "/y" } }`):
- `imp.file` = XML file path
- `imp.attributes` = `map_to` (compiled-class key) and `where` (XPath selector)

Export shape (from LML `export { format xml { file "..."; root "X"; indent "true" } }`):
- `exp.attributes` = `file`, `root` (defaults to `ClassNames`), `indent`, `encoding`

## Integration test

`spec/integration/instance_import_export_spec.rb` verifies the full flow:
LML model definition → ModelCompiler.compile → LML instances file → Pipeline →
Executor.run → XML import → collection validation → XML export.
